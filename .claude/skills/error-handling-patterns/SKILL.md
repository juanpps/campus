# Error Handling Patterns
- NINGUNA pantalla en blanco; usar Skeletons y Error Boundaries en Next.js App Router.
- Todo formulario debe ser validado con Zod antes de enviar.
- Todos los errores de API/UI deben atraparse y mostrarse gentilmente vía Toast (ej: "Raptorcito, ocurrió un error...").
